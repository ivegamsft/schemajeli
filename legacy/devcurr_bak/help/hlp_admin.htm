Servers
The Servers Administration screen can be used to enter and update Server meta-data.
The following fields are displayed on the Server Administration screen.

  • Created By: The user id of the user that entered the server into the system.
  • Create Date: The date that the server meta-data was entered into the system.
  • Last Modified By: The user id of the user that last updated the server meta-data
  • Last Modified: The data that the server meta-data was last updated
  • Server Name: The Navistar Server name.
  • Operating System: The Industry name for the Operating system that resides on the server
  • Operating System Version: The vendor's version number of the Operating system
  • Location: The Navistar location where the server resides
  • Description: A short description of the server.

    Standard Abbreviations
    The Standard Abbreviations administration screen can be used to enter and update Standard Abbreviation meta-data.
    The following fields are displayed on the Standard Abbreviations Administration screen.

  • Created By: The user id of the user that entered the standard abbreviation into the system.
  • Create Date: The date that the standard abbreviation meta-data was entered into the system.
  • Last Modified By: The user id of the user that last updated the standard abbreviation meta-data
  • Last Modified: The data that the standard abbreviation meta-data was last updated
  • Standard Abbreviation Source: A character, a word, or a phrase that is the source of a Navistar's Standard Abbreviated Name. The abbreviated name is used to form a component of a data dictionary name and a DB2 name. For example: Source: 'Antilock Braking System'; Standard Abbreviation: 'ABS'
  • Abbreviation: A Navistar-defined standard abbreviated version of a word or a phrase that can be used as a key word to search for a particular element name. For example: 'CUST' is the Standard Abbreviated Name for 'Customer'.
  • Class Word Indicator: The Class Word Indicator is used to indicate the component of an element name as a class word. Possible values are 'Y' or 'N'. All element names used in Navistar contain PRIME, and/or MODIFIER, and CLASS WORD. Examples of class words are such as: AMT for Amount, Cost for Cost, CD for Code, NME for Name, etc.

    Users
    The Users Administration screen can be used to enter update and delete User meta-data.
    The following fields are displayed on the Users Administration screen.

  • Created By: The user id of the user that entered the user meta-data into the system.
  • Create Date: The date that the user meta-data was entered into the system.
  • Last Modified By: The user id of the user that last updated the user meta-data
  • Last Modified: The data that the user meta-data was last updated
  • User Id: Navistar - issued user id.
  • Authority: The level of security that is issued to the user. Possible values are: 'USER' - has access to SEARCH and REPORTING functions; 'MAINTAIN' - has access to SEARCH, REPORTING and MAINTENANCE functions; 'ADMINISTER' - has access to SEARCH, REPORTING, MAINTENANCE and ADMINISTRATION functions.
  • User First Name: The First Name of the user has to match the one in Outlook's e-mail address. If it doesn't match the one in e-mail address, the DDL will not be mailed to the user.
  • User Last Name: The Last Name of the user has to match the name in Outlook's e-mail address.
