Relational Object

    A Relational Object is a top-level objects that the Navistar Repository stores. For example; Databases are contained in servers therefore servers are the top-level/parent object of databases. All of the objects contained in the repository are stored using the parent-child relationship, and this relationship is enforced by strict rules incorporated within the core of the repository. In some instances, the parent object will not be found. This is due to the lack of data available during the cutover from MSP. In these cases, a "N/A" or null values will be displayed. All new objects entered will be required to follow the parent-child relationship.

Database
The Database maintenance screen can be used to enter, delete and update databases' meta-data.
The following fields are displayed on the Database maintenance screen.

  • Server: The Navistar Server name.
  • Database Business Name: A free-form business description of the database.
  • Database Name: The database name, beginning with the ASC code.
  • RDBMS: Vendor's relational database name, for example "Informix"
  • RDBMS Version: Database's latest release/version number.
  • Description: A short description of the database.

    Table
    The Table maintenance screen can be used to enter and update tables' meta-data.
    The following fields are displayed on the Table maintenance screen.
  • Table Name: The Table Name, beginning with the ASC Code.
  • Status: The current status code of the table i.e. DVLP=Development
  • Database: The name of the database that the table belongs to. This field will not be validated unless it's keyed in. Please enter the database meta-data through the Database Maintenance screen.
  • Description: A short description of the table.
  • Move/Remove: Checkboxes to tag the elements to be moved to another location within the table or to remove an alement from the table.
  • Insert After: Checkboxes to tag the insertion point for adding new elements or moving the elements within the table.
  • Elements: A list of elements that are associated with the table.
  • Key: A number for the primary key of the table. This number will be used to generate an index within the DDL.
  • Status: The current status code of the element i.e. DVLP= In Development.

    Table Maintenance Screen Navigation

    Elements can be added to a table by entering search criteria or a full element name into the "SEARCH" box. If search criteria is too general, the screen will prompt to enter a more specific one.
    By default, the elements are added to the back of the table, but the insertion point can be specified by checking off the "Insert After" box next to the element after which the new element is to be inserted.
    An element can be removed from the table by checking off the box next to the element to be removed and clicking on "Move/Remove" button.
    An element or a group of elements can be moved to a new location within the table by checking off the element(s) to be moved in the "Move/Remove" column and clicking on the insertion point in the "Insert After" column.

    Changes to table meta-data WILL NOT be applied unless the "DONE" button is clicked.

    Element

    The Element maintenance screen can be used to enter update and delete element meta-data. When elements are being created, a developer enters a business name for the element. A business name consists of a term/prime word and a class/modifier word. All elements must include a class word. When the developer clicks submit, a dictionary name, and a column name is generated from the repository. The dictionary name consists of the terms entered in the business name in their abbreviated form, separated by hyphens. The column name is also the abbreviated business name however, it is separated by underscores. If the column name is longer than the allowed 18 character limit, the developer will be prompted to shorten the column name name.
    The following fields are displayed on the Element maintenance screen:

    • Status: The current status code of the element i.e. DVLP= In Development
    • Business Name: The full business name of an element.
    • Dictionary Name: The abbreviated business name, separated by hyphens.
    • Column Name: The abbreviated business name, separated by underscores, that is used to generate the DDL report. The column name has less characters than the dictionary name, because of the 18 character limit to column names designated by DB2.
    • Data Type: The data type of the element.
    • Length: The logical length of the element.
    • Decimal Length: If the data type requires decimal places, this is the number of places used by the element.
    • Description: A short description of the element.
