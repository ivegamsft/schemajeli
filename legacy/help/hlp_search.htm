Databases

This search option provides a way to find database meta-data contained within the repository. By entering the first few characters of the database name and clicking submit, the query will return all of the rows that match your criteria. This query uses the 'LIKE' operator and a '%' is appended to the query in order to find an appropriate match.

The fields returned are:

  • names of all of the databases that match your criteria
  • status each database
  • the name of the server the databases is currently hosted on
  • the RDBMS, i.e. Informix or DB2
  • the current version of the RDBMS

Tables

This search option provides a way to find table meta-data contained within the repository. By entering the first few characters of the table name and clicking submit, the query will return all of the rows that match your criteria. This query run the 'LIKE' operator and a '%' is appended to the query in order to find an appropriate match.

The fields returned are:

  • names of all of the tables that match your criteria
  • status each table
  • the user id of the person who first entered the table into the repository
  • the user id of the person who last modified the table

Elements

This search option provides a way to find element meta-data contained within the repository. By entering the first few characters of the element business name and clicking submit, the query will return all of the rows that match your criteria. This query uses the 'LIKE' operator and a '%' is appended to the query in order to find an appropriate match. Use of the wildcard character is allowed elsewhere in the search criteria.

The fields returned are:

  • the dictionary name of each element that matched your criteria
  • the business name of of each element that matched your criteria
  • the status each element
  • the version of each element

Element Key Words

This search option provides a way to find element meta-data contained within the repository. By entering key words and clicking submit, you can search for elements that contain the key word combination entered. For example, you can search for all elements that contain 'Customer' and 'Name'. This query uses the 'AND' operator and may take longer than other queries since it will search all of the elements that contain the criteria somewhere in the business name.

The fields returned are:

  • the dictionary name of each element that matched your criteria
  • the business name of of each element that matched your criteria
  • the status each element
  • the version of each element

Standard Abbreviations

This search option provides a way to find the standard abbreviation for a given word or phrase or a vice-versa. By entering an abbreviation or a word and clicking submit, you will receive a result set that includes:

  • the full word or phrase that matches your criteria
  • the standard abbreviation for each word or phrase
  • and whether or not that abbreviation can be used as a class/modifier word in a business name
