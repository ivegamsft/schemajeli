Relational Object

    A Relational Object is a top-level objects that the Navistar Repository stores. For example; Databases are contained in servers therefore servers are the top-level/parent object of databases. All of the objects contained in the repository are stored using the parent-child relationship, and this relationship is enforced by strict rules incorporated within the core of the repository. In some instances, the parent object will not be found. This is due to the lack of data available during the cutover from MSP. In these cases, a "N/A" or null values will be displayed. All new objects entered will be required to follow the parent-child relationship.

Server
A summary report for a server will display the following meta-data for the selected server:

  • Server Name: The Navistar Server name
  • No of Databases: The number of databases contained in this server
  • OS Name: The operating system of the server i.e. UNIX or Windows NT
  • OS Version: The current version of the server operating system
  • Location Code: The physical location code of the server i.e. BRK = Brookfield
  • Created By: The user id of the user that entered the server into the system
  • Last Modified: The data that the server meta-data was last updated
  • Last Modified By: The user id of the user that last updated the server meta-data
  • Description: A short description of the server

      Children

    • Database Name: The ASC code name for the database
    • Database Description: A short description of the database
    • Status: The current status code of the database i.e. DVLP=Development
    • No of Tables: The number of tables this database contains

    Element
    A detail report for an element will display the following meta-data for the selected element:

  • Element Business Name: The full business name of an element
  • Status: The current status code of the element i.e. DVLP= In Development
  • No of Tables Where Used: The table name status, where the element is currently being used
  • Created By: The user id of the user that entered the server into the system
  • Last Modified: The data that the server meta-data was last updated
  • Last Modified By: The user id of the user that last updated the server meta-data
  • Element Dictionary Name: The abbreviated business name, separated by hyphens
  • Element Column Name: The abbreviated business name, separated by underscores, that is used to generate the DDL report. The column name has less characters than the dictionary name, because of the 18 character limit to column names designated by DB2
  • Data Type: The data type of the element
  • Length: The logical length of the element
  • Decimal Length: If the data type requires decimal places, this is the number of places used by the element
  • Description: A short description of the element

      Tables Where Used

    • Table Name: The table name where this element is currently being used
    • Status: The current status code of the table i.e. DVLP=Development
